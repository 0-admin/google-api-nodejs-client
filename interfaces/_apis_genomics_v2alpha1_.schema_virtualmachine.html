bootDiskSizeGb

bootDiskSizeGb: number
  • Defined in apis/genomics/v2alpha1.ts:708

The size of the boot disk, in gigabytes. The boot disk must be large enough to accommodate all of the docker images from each action in the pipeline at the same time. If not specified, a small but reasonable default value is used.

bootImage

bootImage: string
  • Defined in apis/genomics/v2alpha1.ts:719

The host operating system image to use. At present, only Container Optimized OS images may be used. The default value is "projects/cos-cloud/global/images/family/cos-stable" which selects the latest stable release of Container Optimized OS. This option is provided to allow testing against the beta release of the operating system to ensure that the new version does not interact negatively with production pipelines. To test a pipeline against the beta release of COS, use the value "projects/cos-cloud/global/images/family/cos-beta".

cpuPlatform

cpuPlatform: string
  • Defined in apis/genomics/v2alpha1.ts:729

The CPU platform to request. An instance based on a newer platform may be allocated but never one with less capabilities. The value of this parameter must be a valid Compute Engine CPU platform name (such as "Intel Skylake"). This parameter is only useful for carefully optimized work loads where the CPU platform has a significant impact. For more information about the effect of this parameter, please visit https://cloud.google.com/compute/docs/instances/specify-min-cpu-platform.

disks

disks: Schema$Disk[]
  • Defined in apis/genomics/v2alpha1.ts:733

The list of disks to create and attach to the VM.

labels

labels: any
  • Defined in apis/genomics/v2alpha1.ts:741

Optional set of labels to apply to the VM and any attached disk resources. These labels must adhere to the name and value restrictions on VM labels imposed by Compute Engine. These labels are applied at creation time to the VM and are applied on a best-effort basis to attached disk resources shortly after VM creation.

machineType

machineType: string
  • Defined in apis/genomics/v2alpha1.ts:747

The machine type of the virtual machine to create. Must be the short name of a standard machine type (such as "n1-standard-1") or a custom machine type (such as "custom-1-4096").

network

preemptible

serviceAccount

The service account to install on the VM. This account does not need any permissions other than those required by the pipeline.
